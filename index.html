Prmeros pasos en Git y Github

¿Git? ¿Github?

Git es una herramienta de control de versiones (otra vez lo mismo) es decir, es una herramienta que nos ayuda a los programadores a no tener que preocuparnos por hacer copias del software que estemos desarollando, si algo se te rompió en el proceso no importa, podrás regresar a una versión anterior del programa donde todo funconaba bien

Git nos permite guardar y regresar tantas veces queramos, y gracias a Microsoft podemos compartir código con otros desarolladores y tener un lugar donde subir todo el código y consultarlo para todos los programdores, gracias a estas plataformas durante la pandemia todos los programadores hemos podido coadyuvar a la regeneración de la vida pública, hacer respiradores y sitios web para poder facilitar el trabajo en linea

Git es una ventana de comandos, Github es una página web, no son lo mismo.
Veamos a git a partir de ahora como nuestro amigo y un personaje secundario que va a estar al pendiente y memorizar todos los cambios en nuestro código TQM Octocat

Git
  • Gestiona archivos en el computador
  • Permite crear versiones distintas
  • Puede subir el codigo a internet
  • Crea el ambiente necesario para el desarollo en colaboración
  • Permite saltar entre distintas versiones
  • Se instala en el computador
Github
  • Gestiona archivos finales en la web
  • Permite crear versiones distintas
  • El código necesita estar subido a internet
  • Es el resultado de crear el ambiente necesario para el desarollo en colaboración
  • Permite saltar entre distintas versiones
  • Se consulta en la web

Instalación

Para poder realizar la instalación de Git necesitamos seguir los siguientes pasos:

  1. Entrar a la página oficial de git y ubicar la descarga adecuada para tu sistema
  2. Descargar el paquete deseado para la distribución de sistema operativo que tienes
  3. Ejecutar el instalador (windows) o ejecutar el comando sudo dpkg -i [nombre del archivo] (en Debian)

Para poder crear una cuenta de Github necesitamos seguir los siguientes pasos

  1. Entrar a la página oficial de Github y ubicar el botón "registrarse"
  2. Seguir los pasos y completar los formularios
  3. Confirmar el correo electrónico

Comandos de Git from zero to hero

Antes que nada tenemos que dejar en claro las tres areas de trabajo de git y como interactúan entre ellas.

¿Que?¿Area de trabajo?
Pues si, un area de trabajo es como un escritorio, ponemos ahi las cosas que necesitamos, ya sea que estén organizadas o desorganizadas.
Regresando al punto, las tres areas de trabajo son:

  • Working directory: Esta parte se puede generalizar como tu carpeta de siempre, en la carpeta llamada ver2.3.4.5final está tu código más reciente, el cual es invaluable e insuperable, pero nuestro querido Octocat no sabe que existe para informarle a Octocat que existe debemos "mover" nuestro código al staging area (descuida, todo va a seguir ahi).
  • Staging area: Aquí, nuestro pequeño amigo Octocat ya está informado de la existencia de todos los archivos que hayas seleccionado, ya está al tanto de sus modificaciones y su contenido sin embargo aun no le decimos que los salve o que cree un checkpoint para poder regresar a el si borramos el código en este punto Octocat va a saber que algo cambió, o que un archivo no está más asi que debemos decirle que lo salve en un .git
  • .git directory: Acá nuestro código ya está salvado y podemos regresar a el cuantas veces necesitemos incluso si borramos los archivos octocat está al pendiente de ellos y sabe qué borró, que se modificó y que se agregó, el sabe todo lo que suceda con esos archivos, Octocat es poderoso.

Working
Area de trabajo

Es el lugar específico donde trabajas, no necesitas ubicarlo ni reconocerlo ya que siempre has estado trabajado en el.

Staging
Sala de espera

Es un lugar hipotético donde informas al sistema Git de que los archivos existen y ponga atención a sus cambios.

.git
Checkpoint

Es un lugar hipotético donde mediante un "commit" salvaste tu código con un checkpoint, es el lugar donde puedes regresar.
